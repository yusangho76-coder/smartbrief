# UAM 운항 브리핑 시스템 사업계획서 (초기창업패키지 딥테크 특화형 양식 작성안)

## 1. 명칭
**UAMBrief (Urban Air Mobility Briefing System)**
- 범주: 인공지능프로그램 (AI 기반 항공운항 지원 시스템)

---

## 2. 아이템 개요

### 제품·서비스 개요

**사용 용도**
- UAM(도심항공교통) 운항 전, 조종사 및 운항관리자에게 필수적인 NOTAM(항공정보), 기상정보, 도심 장애물 정보를 통합 분석하여 안전 중심의 운항 브리핑을 자동 생성하는 시스템
- 웹 기반 대시보드 형태로 제공되며, 운항 경로 입력 시 실시간으로 위험요소를 분석하여 브리핑 리포트를 생성
- 버티포트(수직이착륙장) 간 이동 시 필요한 모든 운항 정보를 한눈에 확인 가능

**사양**
- 플랫폼: 웹 기반 클라우드 서비스 (SaaS)
- 입력 데이터: 
  - 운항 경로/구간 정보 (출발·도착 버티포트, 경유지)
  - NOTAM (항공정보공고) 데이터
  - 기상 데이터 (METAR/TAF, 저고도 바람/난류, SIGMET)
  - 도심 장애물/건물 데이터 (3D 지형 정보)
  - 버티포트 운영 상태 데이터
- 출력 형태: 
  - 운항 브리핑 리포트 (PDF/웹 형식)
  - 위험요소 하이라이트 및 우선순위 표시
  - 운항 가능성 판단 보조 정보
  - 체크리스트 자동 생성

**가격**
- B2B SaaS 모델: UAM 운영사별 월 구독료 (사용자 수 및 노선 수 기반)
- 예상 가격: 월 200만원 ~ 500만원 (기본 플랜 기준, API 추가 이용 시 별도 과금)

### 핵심 기능·성능

**핵심 기능**
1. **저고도 운항 위험요소 통합 분석**
   - 도심 환경 특화: 저고도(100~500m) 기상 정보, 건물 간 바람/난류 예측
   - 장애물 영향 평가: 운항 경로 상의 고층건물, 송전선, 통신탑 등 영향 분석
   - 공역 제한 자동 감지: 비행 금지구역, 일시적 공역 제한, 버티포트 운영 제약 자동 식별

2. **AI 기반 브리핑 자동 생성**
   - Google Gemini API 기반 자연어 생성 엔진 활용
   - 운항 단계별(이륙, 순항, 착륙) 위험요소 자동 분류 및 우선순위화
   - 조종사 친화적 문장으로 위험요소 설명 및 대응 방안 제시

3. **실시간 데이터 통합 및 업데이트**
   - NOTAM 데이터 실시간 수집 (항공공단 API 연동)
   - 기상 데이터 자동 갱신 (METAR/TAF, 저고도 기상 모델)
   - 버티포트 운영 상태 실시간 반영

**성능 지표**
- 브리핑 생성 시간: 평균 10초 이내 (기존 수동 브리핑 30분 → 자동화)
- 위험요소 탐지율: 95% 이상
- 위험요소 누락률: 5% 이하
- 브리핑 정확도: 운항관리사 평가 4.5/5.0 이상 목표

### 고객 제공 혜택

1. **운항 안전성 향상**
   - 중요 위험요소 누락 최소화로 안전사고 예방
   - 표준화된 브리핑 체계로 품질 편차 해소
   - 위험도 기반 우선순위 제시로 의사결정 지원

2. **운영 효율성 제고**
   - 브리핑 시간 단축 (30분 → 10초)으로 운항 준비 시간 절약
   - 운항관리사 업무 부담 감소로 인건비 절감 가능
   - 24시간 자동화 서비스로 야간/비상 상황 대응력 향상

3. **규제 대응 능력 강화**
   - UAM 규제 변화에 따른 브리핑 요구사항 자동 반영
   - 운항 기록 자동 보관으로 규제 준수 증빙 자료 확보
   - UTM(무인항공기 교통관리) 시스템 연동 가능 구조

## 3. 문제 인식

### 국내·외 시장 현황 및 문제점

**시장 현황**
- UAM 시장 급성장: 국내 2025년 상용화 목표, 2035년 전 세계 시장 규모 약 1조 달러 예상
- 글로벌 주요국(미국, EU, 일본) UAM 상용화 추진 중
- 국내: 서울 강남-김포 등 주요 노선, 서울대공원, 여의도 등 버티포트 구축 계획 발표

**현재 시장의 문제점**
1. **저고도·도심 환경의 복잡한 위험요소**
   - 기존 항공 브리핑 시스템은 고고도 항로 중심으로 설계되어 저고도(100~500m) 운항에 적합하지 않음
   - 도심 기상 변화 급속: 건물 간 바람, 열섬 현상, 국지적 강풍 등 예측 어려움
   - 장애물 밀집: 고층건물, 송전선, 통신탑 등 실시간 영향 평가 필요

2. **표준화된 브리핑 체계 부재**
   - UAM 상용화 초기 단계로 운항 브리핑 표준 절차 미정립
   - 각 운영사별로 수동 브리핑에 의존하여 품질 편차 발생
   - 운항관리사 숙련도에 따라 브리핑 품질 차이 심각

3. **다양한 데이터 소스의 분산 및 통합 어려움**
   - NOTAM, 기상정보, 버티포트 운영상태, 장애물 정보 등 다중 소스
   - 각 정보를 개별적으로 확인하여 통합 판단에 시간 소요
   - 실시간 업데이트 정보 누락 가능성

4. **기존 시스템의 한계**
   - 기존 항공 기상/NOTAM 서비스: UAM 운항 시나리오 미고려
   - 도심 교통 정보 시스템: 항공 운항 정보 부재
   - 항공사 내부 브리핑 툴: 고고도 항로 중심, UAM 특화 기능 없음

### 개발 필요성

1. **UAM 상용화 초기 안전 체계 구축**
   - 새로운 운항 환경(UAM)에서 안전사고를 예방하기 위한 표준화된 브리핑 시스템 필수
   - 상용화 초기에 안전 사고 발생 시 전체 산업에 미치는 영향 심각 → 사전 예방 체계 필요

2. **운영 효율성 및 경쟁력 확보**
   - UAM 운영사들은 운항 안전성과 운영 효율성 확보를 통해 시장 경쟁력 강화 필요
   - 자동화된 브리핑 시스템으로 인건비 절감 및 서비스 품질 향상

3. **규제 요구사항 선제 대응**
   - 항공안전법, UAM 관련 규제 강화 예상 → 규제 준수를 위한 체계적 브리핑 기록 관리 필요
   - UTM(무인항공기 교통관리) 시스템과의 연동 필요성 증가

4. **글로벌 시장 진출 기반 마련**
   - 국내 기술로 UAM 브리핑 시스템을 선도적으로 개발하여 글로벌 시장 진출 기회 창출
   - 해외 UAM 운영사들의 국내 기술 도입 가능성

## 4. 실현 가능성

### 개발하고자 하는 창업 아이템을 사업기간 내 제품·서비스로 개발 또는 구체화 하고자 하는 계획

**최종 산출물 형태 및 수량**

**Phase 1 (0~6개월): MVP 개발**
- 산출물: UAM 브리핑 시스템 MVP (웹 기반)
  - 기능: 기본 NOTAM/기상 통합 분석, 브리핑 자동 생성
  - 운영 범위: 제한된 도심 노선 3개, 버티포트 5개
  - 파일럿 운영: UAM 운항사 1개사와 협력 테스트
  - 수량: MVP 1건, 파일럿 운영 보고서 1건

**Phase 2 (6~12개월): 기능 고도화 및 베타 서비스**
- 산출물: UAM 브리핑 시스템 베타 버전
  - 기능: 저고도 기상/장애물 모델 고도화, 위험도 스코어링 고도화
  - 운영 범위: 확장 노선 10개, 버티포트 15개
  - 베타 고객: UAM 운항사 3개사
  - 수량: 베타 버전 1건, 베타 운영 보고서 1건, 사용자 평가 보고서 1건

**Phase 3 (12~24개월): 상용화 및 확장**
- 산출물: UAM 브리핑 시스템 상용 버전
  - 기능: UTM 연동, 다국어 지원, 맞춤형 브리핑 템플릿
  - 운영 범위: 전국 주요 노선, 버티포트 30개 이상
  - 상용 고객: UAM 운항사 5개사 이상
  - 수량: 상용 버전 1건, 기술 문서화 자료 1건, API 문서 1건

**기술 개발 일정**
- 0~2개월: UAM 브리핑 템플릿 설계, 데이터 수집 파이프라인 구축
- 2~4개월: AI 기반 브리핑 생성 엔진 개발, 저고도 기상 분석 모델 개발
- 4~6개월: 웹 대시보드 개발, MVP 통합 테스트 및 파일럿 운영
- 6~9개월: 저고도 기상/장애물 모델 고도화, 성능 최적화
- 9~12개월: 베타 서비스 운영 및 피드백 반영
- 12~18개월: UTM 연동 개발, 다국어 지원, 맞춤형 기능 개발
- 18~24개월: 상용화 및 고객 확대

### 차별성 및 경쟁력 확보 전략

**차별성 (vs. 기존 시스템)**

1. **UAM 운항 특화 설계**
   - 기존 항공 브리핑 시스템: 고고도(FL200 이상) 항로 중심
   - 본 시스템: 저고도(100~500m) 도심 운항 특화, 건물 간 바람/난류 분석

2. **AI 기반 브리핑 자동 생성**
   - 기존 시스템: 데이터 단순 제공 (METAR/TAF, NOTAM 원문)
   - 본 시스템: AI가 위험요소를 분석하여 조종사 친화적 문장으로 브리핑 자동 생성

3. **도심 장애물 통합 분석**
   - 기존 시스템: 장애물 정보 부재
   - 본 시스템: 고층건물, 송전선, 통신탑 등 3D 지형 정보와 운항 경로 교차 분석

4. **실시간 통합 분석**
   - 기존 시스템: 다중 데이터 소스를 사용자가 개별 확인
   - 본 시스템: NOTAM, 기상, 버티포트, 장애물 정보를 실시간 통합 분석하여 일괄 제공

**경쟁력 확보 전략**

1. **기술적 차별화**
   - Google Gemini API 기반 고도화된 자연어 생성 엔진 활용
   - 저고도 기상 예측 모델과 장애물 영향 분석 알고리즘 자체 개발
   - UAM 운항 데이터 축적을 통한 모델 정확도 지속 개선

2. **파트너십 및 데이터 확보**
   - 항공공단과의 NOTAM 데이터 API 협약
   - 기상청 저고도 기상 데이터 활용 협의
   - UAM 운영사와의 파일럿 운영을 통한 실전 데이터 축적 및 피드백 반영

3. **규제 선제 대응**
   - UAM 관련 규제 변화를 실시간 반영하는 모듈화된 시스템 구조
   - 운항 기록 자동 보관 및 규제 준수 증빙 자료 생성 기능 내장

4. **시장 선점 전략**
   - 국내 UAM 상용화 시기(2025년) 전 MVP 완성하여 시장 선점
   - 파일럿 운영 성과를 바탕으로 글로벌 시장 진출 준비

5. **확장성 및 연동성**
   - UTM(무인항공기 교통관리) 시스템과의 API 연동 구조 설계
   - 다국어 지원 및 해외 시장 진출 가능한 확장 구조

**지속적 경쟁력 유지 방안**
- 운항 데이터 축적을 통한 AI 모델 지속 학습 및 개선
- 사용자 피드백 기반 기능 개선 및 맞춤형 서비스 제공
- 산학연 협력 연구를 통한 최신 기술 동향 반영
