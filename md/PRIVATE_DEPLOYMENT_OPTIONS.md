# 소스 코드 공개 없이 배포하는 방법

소스 코드를 공개하지 않고 사용자에게 배포하는 여러 방법을 설명합니다.

---

## 🔒 문제점

**현재 배포 방법의 문제:**
- 사용자에게 전체 소스 코드 제공 필요
- 소스 코드가 공개됨
- 비즈니스 로직 노출
- API 키 등 민감 정보 노출 위험

---

## 💡 해결 방법

### 방법 1: GitHub Actions 자동 배포 (추천) ⭐

**개념:**
- 소스 코드는 GitHub에만 있음 (비공개 저장소)
- 사용자는 GitHub 저장소에 접근 권한만 받음
- GitHub Actions가 자동으로 배포
- 사용자는 소스 코드를 직접 볼 필요 없음

**장점:**
- ✅ 소스 코드 보호
- ✅ 자동 배포
- ✅ 사용자는 설정만 하면 됨
- ✅ 코드 업데이트 시 자동 반영

**단점:**
- ⚠️ GitHub 저장소 접근 권한 필요
- ⚠️ GitHub Actions 설정 필요

**구현 방법:**

1. **GitHub에 비공개 저장소 생성**
2. **GitHub Actions 워크플로우 설정** (이미 있음)
3. **사용자에게 저장소 접근 권한 부여** (Collaborator)
4. **사용자는 Secrets만 설정**
   - GCP_PROJECT_ID
   - GCP_SA_KEY
   - GEMINI_API_KEY
5. **자동 배포 완료**

**사용자 관점:**
```
1. GitHub 저장소 초대 받기
2. Settings → Secrets에 값 입력
3. Actions 탭에서 배포 실행
4. 완료! (소스 코드는 안 봄)
```

---

### 방법 2: Docker 이미지만 제공

**개념:**
- 소스 코드를 Docker 이미지로 빌드
- Docker 이미지만 사용자에게 제공
- 사용자는 이미지만 배포

**장점:**
- ✅ 소스 코드 직접 노출 없음
- ✅ 이미지만 배포

**단점:**
- ⚠️ Docker 이미지 역컴파일 가능 (완전한 보호 아님)
- ⚠️ 이미지 파일 크기 큼
- ⚠️ 업데이트 시 새 이미지 제공 필요

**구현 방법:**

```bash
# 1. Docker 이미지 빌드
docker build -t smartnotam:latest .

# 2. 이미지 저장
docker save smartnotam:latest | gzip > smartnotam.tar.gz

# 3. 사용자에게 이미지 파일 제공
# 사용자는 이미지만 받아서 배포
```

**사용자 관점:**
```
1. Docker 이미지 파일 받기
2. 이미지 로드: docker load < smartnotam.tar.gz
3. Cloud Run에 배포
```

### 방법 3: SaaS 형태로 제공 (가장 안전)

**개념:**
- 개발자가 직접 배포 및 운영
- 사용자는 URL만 받아서 사용
- 완전한 소스 코드 보호

**장점:**
- ✅ 완전한 소스 코드 보호
- ✅ 사용자는 배포 불필요
- ✅ 자동 업데이트
- ✅ 중앙 관리

**단점:**
- ⚠️ 개발자가 운영 비용 부담
- ⚠️ 사용자별 설정 관리 필요

**구현 방법:**

1. **개발자가 Cloud Run에 배포**
2. **사용자별 환경변수 설정**
   - 각 사용자의 GEMINI_API_KEY
3. **사용자에게 URL 제공**
4. **사용자는 URL만 사용**

**사용자 관점:**
```
1. 개발자에게 요청
2. URL 받기
3. 바로 사용! (배포 불필요)
```

### 방법 4: 컴파일된 바이너리 (Python은 어려움)

**개념:**
- Python 코드를 바이너리로 컴파일
- .pyc 파일 또는 PyInstaller 사용

**장점:**
- ✅ 소스 코드 직접 노출 없음

**단점:**
- ❌ Python은 완전한 컴파일 어려움
- ❌ 역컴파일 가능
- ❌ 복잡함

**비추천:** Python 특성상 완전한 보호 어려움

## 🔧 GitHub Actions 자동 배포 설정 (상세)

### 1. GitHub 비공개 저장소 생성

1. GitHub에서 "New repository"
2. "Private" 선택
3. 저장소 생성

### 2. 워크플로우 확인

`.github/workflows/deploy-cloud-run.yml` 파일이 이미 있습니다.

### 3. 사용자에게 권한 부여

**Settings → Collaborators → Add people**

### 4. 사용자가 Secrets 설정

**Settings → Secrets and variables → Actions → New repository secret**

필요한 Secrets:
- `GCP_PROJECT_ID`: Google Cloud 프로젝트 ID
- `GCP_SA_KEY`: 서비스 계정 키 JSON
- `GEMINI_API_KEY`: Gemini API 키

### 5. 배포 실행

**Actions 탭 → "Deploy to Google Cloud Run" → Run workflow**

## ⚠️ 주의사항

### GitHub Actions 사용 시

1. **비공개 저장소 필수**
   - 공개 저장소는 누구나 볼 수 있음

2. **Secrets 보안**
   - Secrets는 암호화되어 저장
   - 로그에 출력되지 않음

3. **접근 권한 관리**
   - 필요한 사람만 Collaborator 권한
   - 필요 없으면 제거

### Docker 이미지 사용 시

1. **완전한 보호 아님**
   - 이미지에서 파일 추출 가능
   - 하지만 일반 사용자에게는 충분

2. **이미지 크기**
   - 수백 MB ~ 수 GB
   - 전송 시간 고려
