# saved_results 폴더 동작 방식 분석

## 현재 동작 방식

### 1. HTML 저장 버튼 클릭 시
1. 사용자가 "HTML 저장" 버튼 클릭
2. 현재 페이지의 전체 HTML을 JavaScript로 가져옴
3. 서버에 POST 요청으로 HTML 전송 (`/save_html` API)
4. 서버가 `saved_results/` 폴더에 파일 저장
5. **동시에** 브라우저에서 자동 다운로드 시작 (`/download_html/<filename>`)
6. 사용자의 로컬 컴퓨터/태블릿에 HTML 파일이 다운로드됨

### 2. 오프라인 사용 시나리오

#### ✅ 가능한 방법 (현재 구현):
1. **온라인 상태에서 미리 다운로드**
   - 비행 전에 "HTML 저장" 버튼 클릭
   - HTML 파일이 로컬에 다운로드됨
   - 비행 중 오프라인 환경에서 다운로드된 로컬 파일 열기

#### ❌ 불가능한 방법:
- 서버의 `saved_results/` 폴더에 직접 접근 (오프라인에서는 불가능)
- 비행 중 온라인 상태로 서버에서 재다운로드 (인터넷 연결 필요)

---

## 문제점 분석

### 현재 구현의 한계
1. **서버 저장은 사실상 백업/재다운로드 용도**
   - 온라인 상태에서만 접근 가능
   - 오프라인에서는 서버 파일에 접근 불가

2. **실제 오프라인 사용은 로컬 다운로드에 의존**
   - 사용자가 버튼을 클릭해야만 다운로드됨
   - 자동으로 로컬에 저장되지 않음

3. **saved_results 폴더의 실제 용도**
   - ✅ 온라인 상태에서 재다운로드
   - ✅ 백업/아카이브
   - ❌ 오프라인 직접 접근 (불가능)

## 개선 방안

### 옵션 1: 서버 저장 기능 제거 (권장)
**이유:**
- 오프라인 사용 목적이라면 서버 저장이 불필요
- 용량 절감 (8.7MB → 0MB)
- 클라이언트 측에서만 Blob으로 다운로드

### 옵션 2: 자동 다운로드만 유지
**방법:**
- 서버 저장 제거
- 브라우저에서만 Blob URL로 다운로드
- 더 간단하고 효율적

### 옵션 3: 서버 저장 유지 (현재 방식)
**장점:**
- 온라인 상태에서 재다운로드 가능
- 백업 역할

**단점:**
- 서버 용량 사용 (8.7MB)
- 오프라인에서는 접근 불가 (사용자 혼란 가능)

---

## 결론

**saved_results 폴더는 오프라인 직접 접근용이 아니라:**
- 온라인 상태에서의 재다운로드용
- 백업/아카이브용

**실제 오프라인 사용:**
- 사용자가 "HTML 저장" 버튼 클릭 시 브라우저에서 자동 다운로드
- 다운로드된 로컬 파일을 오프라인에서 열어서 사용

**권장 사항:**
- 서버 저장 기능 제거 (용량 절감)
- 브라우저에서만 Blob 다운로드 방식으로 변경
