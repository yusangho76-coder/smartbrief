# OFP 기상 차트 분석 테스트 결과 및 정확도 결론

## 테스트 개요

- **목적**: Vision/OpenCV 없이 **좌표 기반 경로 + 경로 상 픽셀 색상 샘플링**만으로 SigWx 차트 분석이 가능한지, 실제 PDF로 검증.
- **방법**: `test/test_sigwx_color_sampling.py` — flight_data 좌표 → PDF 페이지 텍스트로 bounds 추출 → (lat,lon)→픽셀 변환 → 구간별 샘플 픽셀 RGB → WAFS 표준 색상(SEV/MOD/LGT/CB/Jet) 분류.
- **사용 PDF**: `uploads/20260309_121300_1420be0b_ImportantFile_16.pdf` (RKSI→CYVR, 51 waypoints).

## 테스트 결과 요약

| 항목 | 결과 |
|------|------|
| Waypoint 좌표 추출 | 51/51 정상 |
| SigWx 페이지 탐지 | 0-based 18페이지 |
| 페이지 텍스트 bounds | lat -40~40, lon 100~180 |
| 경로→픽셀 매핑 | **실제 차트 영역과 불일치** (아래 상세) |

## 원인 분석

1. **좌표→픽셀 매핑**
   - bounds는 페이지 텍스트의 N/E 숫자만으로 min/max lat·lon을 구함. **실제 차트가 그리는 영역(여백, 제목, 범례)** 과 일치하지 않음.
   - 샘플 픽셀: EGOBA (398, **27**), ETP1 (1003, **0**), CYVR (**1162**, 0) → **이미지 상단(y≈0) 및 우측 끝**을 가리킴. 즉 **지도 영역이 아닌 여백/헤더**를 샘플링함.
   - 날짜선 보정으로 CYVR을 우측 끝(1162,0)으로 보냈으나, 그 위치는 흰색(255,255,255)으로 **차트 본문이 아님**.

2. **레이아웃 미반영**
   - 회사 OFP SigWx는 상단 제목, 좌측/우측 범례 등으로 **실제 지도 영역이 (0,0)-(W,H) 전체가 아님**. 지도만의 pixel 영역을 알려면 그리드 라벨 위치 OCR 또는 수동 캘리브레이션이 필요하며, 이전 Vision/pdfplumber 시도에서도 실패 이력이 있음.

3. **결론**
   - **좌표 기반 + 색상 샘플링**만으로는 **차트 레이아웃(여백·그리드 실제 위치)을 알 수 없어** 정확한 픽셀 매핑이 불가.
   - 따라서 **차트 이미지 자동 분석으로는 정확도 확보 불가**.

## 권장 방안: API 기반만 사용 (최고 정확도)

| 구분 | 내용 |
|------|------|
| **OFP 기상 차트 (SigWx/ASC/Cross)** | **참고용 이미지로만 제공** (추출 후 새 창 보기). 자동 분석 **하지 않음**. |
| **터뷸런스·기상 분석** | ① OFP SR 테이블, ② **SIGMET/G-AIRMET API**, ③ Wind/Temp(OFP 텍스트), ④ **WAFS(GFS Ellrod)** 및 Gemini 요약만 사용. |
| **정확도** | 실시간/수치 데이터 기반이라 **차트 이미지 해석보다 신뢰도 높음**. |

이 구성을 **유지**하는 것이 현재로서는 **최고 정확도를 확보하는 방안**입니다.

## 테스트 재현 방법

```bash
# 실제 PDF로 색상 샘플링 테스트
python test/test_sigwx_color_sampling.py uploads/20260309_121300_1420be0b_ImportantFile_16.pdf
```

- 경로 상 샘플이 전부 background로 나오고, JPG 랜덤 샘플에서만 터뷸런스 색이 소량 나오는 것을 확인할 수 있음.
