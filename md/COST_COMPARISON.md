# 배포 플랫폼 비용 비교 (실제 사용 패턴 기준)

## 사용자 사용 패턴

- **사용 빈도**: 한 달에 10회
- **1회당 사용 시간**: 약 3분
- **총 사용 시간**: 약 30분/월
- **Gemini API 비용**: 별도 (플랫폼과 무관)

---

## 1. Google Cloud Run (GCR) 비용

### Cloud Run 가격 구조

#### 무료 티어
- **월 200만 요청 무료**
- **월 360,000 vCPU-초 무료** (1 vCPU × 100시간)
- **월 400,000 GiB-초 무료** (1 GiB × 111시간)

#### 추가 요금 (무료 티어 초과 시)
- **요청**: 100만 요청당 $0.40
- **CPU**: $0.00002400 per vCPU-초
- **메모리**: $0.00000250 per GiB-초
- **네트워크**: $0.12 per GB (첫 10GB 무료)

### 사용자 패턴 기준 계산

#### 시나리오: 한 달에 10회, 1회당 3분

```
요청 수: 10회
처리 시간: 10회 × 3분 = 30분 = 1,800초
리소스: 1 vCPU, 2 GiB
```

#### CPU 비용 계산
```
사용량: 1 vCPU × 1,800초 = 1,800 vCPU-초
무료 티어: 360,000 vCPU-초
─────────────────────────────
과금: 0원 (무료 티어 내)
```

#### 메모리 비용 계산
```
사용량: 2 GiB × 1,800초 = 3,600 GiB-초
무료 티어: 400,000 GiB-초
─────────────────────────────
과금: 0원 (무료 티어 내)
```

#### 요청 비용 계산
```
요청 수: 10회
무료 티어: 2,000,000 요청
─────────────────────────────
과금: 0원 (무료 티어 내)
```

### Cloud Run 총 비용

**월 $0 (완전 무료)** ✅

**이유:**
- 10회 요청은 무료 티어(200만 요청) 내
- 30분 사용은 무료 티어(100시간) 내
- 실제 사용 시간만 과금 (사용하지 않을 때는 비용 없음)

## 2. Railway 비용

### Railway 가격 구조

#### Hobby 플랜
- **구독료**: $5/월
- **크레딧**: $5/월 제공
- **리소스 요금**: 사용량 기반

#### 리소스별 초당 요금
- **Memory**: $0.00000386 per GB per second
- **CPU**: $0.00000772 per vCPU per second

### 사용자 패턴 기준 계산

#### 문제점: Railway는 항상 켜져 있어야 함

Railway는 **Serverless가 아닙니다**. 앱이 켜져 있는 시간만큼 과금됩니다.

#### 시나리오 1: 항상 켜져 있음 (최악의 경우)
```
24시간 × 30일 = 720시간/월
시간당 비용: $0.056
─────────────────────────────
월간 비용: $40.32
크레딧 $5 차감 후: $35.32/월
```

#### 시나리오 2: 하루 8시간만 켜져 있음
```
8시간 × 30일 = 240시간/월
시간당 비용: $0.056
─────────────────────────────
월간 비용: $13.44
크레딧 $5 차감 후: $8.44/월
```

#### 시나리오 3: 하루 1시간만 켜져 있음
```
1시간 × 30일 = 30시간/월
시간당 비용: $0.056
─────────────────────────────
월간 비용: $1.68
크레딧 $5 차감 후: $0 (크레딧으로 충분)
```

### Railway의 문제점

**사용자가 한 달에 30분만 사용하더라도:**
- 앱이 켜져 있어야 접근 가능
- 켜져 있는 동안 계속 과금
- 실제 사용 시간(30분)과 무관하게 비용 발생

**최소 비용:**
- 하루 1시간만 켜져 있어도: 월 $1.68 (크레딧으로 커버 가능)
- 하지만 사용자가 접근할 때마다 켜져 있어야 함

## 4. 결론 및 권장사항

### 사용자 패턴 (한 달에 10회, 30분 사용)

#### ✅ **Cloud Run (GCR) 추천**

**이유:**
1. **완전 무료**: 사용 패턴 기준 월 $0
2. **Serverless**: 사용하지 않을 때 비용 없음
3. **자동 스케일링**: 요청이 있을 때만 실행
4. **무료 티어 넉넉함**: 200만 요청, 100시간까지 무료

#### ❌ **Railway 비추천**

**이유:**
1. **항상 켜져 있어야 함**: 사용하지 않을 때도 비용 발생
2. **비용이 높음**: 최소 월 $8-40
3. **사용 패턴에 맞지 않음**: 가끔 사용하는 경우 비효율적

## 5. 다른 사용 패턴 고려

### 시나리오 A: 자주 사용 (하루 10회, 한 달 300회)

#### Cloud Run
```
요청: 300회 (무료 티어 내)
사용 시간: 300회 × 3분 = 900분 = 15시간
─────────────────────────────
비용: $0 (무료 티어 내)
```

#### Railway
```
항상 켜져 있음: $35-40/월
하루 8시간: $8.44/월
```

**결론**: Cloud Run이 여전히 무료

### 시나리오 B: 매우 자주 사용 (하루 100회, 한 달 3,000회)

#### Cloud Run
```
요청: 3,000회 (무료 티어 내)
사용 시간: 3,000회 × 3분 = 9,000분 = 150시간
─────────────────────────────
비용: $0 (무료 티어 내)
```

**결론**: Cloud Run이 여전히 무료

### 시나리오 C: 거의 항상 사용 (24시간 운영)

#### Cloud Run
```
사용 시간: 720시간/월
무료 티어: 100시간
추가: 620시간
─────────────────────────────
CPU 비용: 620시간 × 1 vCPU × $0.00002400/초 = 약 $53.57
메모리 비용: 620시간 × 2 GiB × $0.00000250/초 = 약 $11.16
─────────────────────────────
총 비용: 약 $64.73/월
```

#### Railway
```
항상 켜져 있음: $35-40/월
```

**결론**: 이 경우 Railway가 더 저렴할 수 있음

## 6. 최종 권장사항

### 사용 패턴별 추천 플랫폼

| 사용 빈도 | 추천 플랫폼 | 이유 |
|----------|------------|------|
| **가끔 사용** (월 10-100회) | **Cloud Run** ✅ | 완전 무료 |
| **자주 사용** (월 100-1,000회) | **Cloud Run** ✅ | 여전히 무료 티어 내 |
| **매우 자주 사용** (월 1,000-10,000회) | **Cloud Run** ✅ | 여전히 무료 티어 내 |
| **24시간 운영** (항상 켜져 있음) | **Railway** 또는 **Cloud Run** | 사용량에 따라 다름 |

### 현재 사용자 (월 10회, 30분) 기준

**✅ Cloud Run (GCR) 강력 추천**

**이유:**
1. **완전 무료**: 월 $0
2. **설정 복잡도**: 초기 설정만 하면 이후 자동
3. **Gemini API 비용**: 플랫폼과 무관 (동일)

**Railway를 사용할 이유가 없음:**
- 비용이 높음 ($8-40/월)
- 사용 패턴에 맞지 않음
- 초기 설정이 더 쉬워도 비용이 문제

## 7. Cloud Run 설정 개선 방안

### 문제점: 초기 설정이 복잡함

**해결책:**
1. **자동화 스크립트 개선**: 더 간단하게 만들기
2. **단계별 가이드 제공**: 초보자도 따라할 수 있게
3. **GitHub Actions 자동 배포**: push만 하면 자동 배포

### 제안: GCR 배포를 더 쉽게 만들기

기존 `deploy_mac.sh` 스크립트를 개선하여:
- 자동으로 필요한 것들을 설정
- 사용자가 최소한의 입력만 하면 됨
- 오류 발생 시 명확한 안내
