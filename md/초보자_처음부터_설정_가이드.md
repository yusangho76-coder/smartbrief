# SmartBrief 처음부터 설정하기 (초보자용)

**이 컴퓨터에서** 아래를 처음부터 하는 분을 위한 가이드입니다.

1. Google Cloud(GCP) 가입  
2. 결제 수단 연결  
3. 프로젝트 만들기  
4. API 키 발급  
5. GitHub 가입  
6. SmartBrief 저장소 Fork  
7. **Fork 이후 GCP에서 할 일** (트리거 연결, 환경 변수)  

순서대로만 따라 하시면 됩니다. 건너뛰지 마세요.

---

# 1. Google Cloud(GCP) 가입

Google Cloud는 Google이 제공하는 클라우드 서비스입니다. SmartBrief 배포와 API 사용에 필요합니다.

## 1-1. 가입 페이지 접속

1. 웹 브라우저(Chrome, Safari 등)를 엽니다.  
2. 주소창에 아래 주소를 입력하고 엔터를 누릅니다.  
   **https://console.cloud.google.com**  
3. **Google 계정으로 로그인** 버튼이 보이면 클릭합니다.  
4. **Gmail 주소**와 **비밀번호**를 입력해 로그인합니다.  
   - Google 계정이 없으면 **계정 만들기**로 먼저 가입합니다.  

## 1-2. 이용약관 동의

1. 처음 들어오면 **이용약관** 또는 **약관 동의** 화면이 나올 수 있습니다.  
2. **동의** 또는 **수락**을 눌러 진행합니다.  
3. **무료 체험 시작** 같은 문구가 나오면, 나중에 결제 연결할 때까지 **무료 한도**만 사용할 수 있다고 안내됩니다.  

가입이 끝나면 **Google Cloud Console** 첫 화면(대시보드)이 보입니다.

---

# 2. 결제 수단 연결 (결제 계정 만들기)

Cloud Run 등 일부 서비스는 **결제 계정**이 있어야 사용할 수 있습니다.  
무료 한도(90일) 안에서 쓰면 **요금이 청구되지 않지만**, 반드시 **결제 수단(카드 등)** 을 등록해야 합니다.

## 2-1. 결제 메뉴 열기

1. 왼쪽 위 **≡ (햄버거 메뉴)** 를 클릭합니다.  
2. 목록에서 **결제** 또는 **Billing** 를 클릭합니다.  
3. **결제 계정 관리** 또는 **Manage billing accounts** 를 클릭합니다.  

## 2-2. 결제 계정 만들기

1. **결제 계정 만들기** 또는 **Create account** 를 클릭합니다.  
2. **국가/지역**: 대한민국 (또는 사용 중인 국가) 선택  
3. **통화**: KRW (원) 선택  
4. **동의** 체크 후 **계정 만들기** 클릭  

## 2-3. 결제 수단 등록

1. **결제 수단 추가** 또는 **Add payment method** 를 클릭합니다.  
2. **신용카드** 또는 **직불카드** 정보를 입력합니다.  
   - 카드 번호, 유효기간, CVC, 청구지 주소 등  
3. **저장** 또는 **Save** 를 클릭합니다.  
4. (선택) **무료 체험 크레딧**이 있으면 그걸 먼저 사용할 수 있습니다.  

> **참고**: 무료 한도 내 사용 시 0원 청구입니다. 사용량이 한도를 넘어갈 때만 요금이 부과됩니다. SmartBrief만 가볍게 쓰면 무료 한도로 충분한 경우가 많습니다.

# 3. 프로젝트 만들기

Google Cloud에서는 모든 작업을 **프로젝트** 단위로 합니다. SmartBrief용 프로젝트를 하나 만듭니다.

## 3-1. 프로젝트 만들기 화면으로 이동

1. 왼쪽 위 **≡** → **홈** 또는 **Dashboard** 로 갑니다.  
2. 화면 **맨 위**에 **프로젝트 선택** 또는 **Select a project** 가 보이면 클릭합니다.  
3. 오른쪽 위 **새 프로젝트** 또는 **New Project** 를 클릭합니다.  

## 3-2. 프로젝트 이름·조직 입력

1. **프로젝트 이름**에 예: `smartbrief-my` 또는 `my-smartbrief` 처럼 구분하기 쉬운 이름을 입력합니다.  
2. **위치**(조직)는 **내 조직 없음** 또는 기본값으로 두어도 됩니다.  
3. **만들기** 또는 **Create** 를 클릭합니다.  
4. 잠시 후 프로젝트가 만들어지면, **프로젝트 선택**을 클릭해 방금 만든 프로젝트를 선택합니다.  

## 3-3. 결제 계정 연결

1. **≡** → **결제** 로 다시 갑니다.  
2. **결제 계정**을 클릭한 뒤, **프로젝트 연결** 또는 **Link a project** 를 선택합니다.  
3. 방금 만든 **프로젝트 이름**을 선택하고 **설정** 또는 **Set account** 를 클릭합니다.  

이제 이 프로젝트에서 사용한 비용은 연결한 결제 수단으로 청구됩니다. (무료 한도 내면 0원)

## 3-4. 프로젝트 ID 확인 (나중에 씀)

1. **≡** → **홈** → **프로젝트 선택** 을 클릭합니다.  
2. 방금 만든 프로젝트를 클릭하면, **프로젝트 ID** 가 보입니다.  
   - 예: `smartbrief-my-123456`  
3. 이 **프로젝트 ID**를 메모장에 적어 둡니다. 배포 스크립트 설정할 때 사용합니다.

# 4. API 키 발급

SmartBrief는 **Gemini API 키**를 사용합니다. (Google 지도 기능을 쓰려면 **Google Maps API 키**도 따로 발급합니다.)

## 4-1. Gemini API 키 (필수)

1. 브라우저에서 **https://aistudio.google.com/apikey** 를 엽니다.  
2. **Google 계정으로 로그인**합니다. (위에서 쓴 것과 같은 계정)  
3. **Create API key** 또는 **API 키 만들기** 를 클릭합니다.  
4. **프로젝트 선택**이 나오면, 3단계에서 만든 **프로젝트**를 선택합니다.  
   - 목록에 없으면 **새 Google Cloud 프로젝트**로 만들기 선택 후 진행  
5. **API 키**가 생성되면 **복사** 버튼으로 전체를 복사합니다.  
   - `AIzaSy...` 로 시작하는 긴 문자열입니다.  
6. 메모장에 붙여 넣어 **안전한 곳에 저장**합니다. 나중에 배포 스크립트의 `API_KEY` 에 넣습니다.  

> **참고**: Gemini API 키와 Google Maps API 키는 **같은 Google 계정**에서 각각 발급해서 함께 사용할 수 있습니다. 둘 다 같은 GCP 프로젝트에 연결해 두면 관리하기 편합니다.

> **주의**: 이 키는 다른 사람에게 보여 주거나 인터넷에 올리지 마세요.

## 4-2. Google Maps API 키 (선택, 지도 기능 쓸 때만)

1. **https://console.cloud.google.com** 에서 **방금 만든 프로젝트**가 선택된 상태로 갑니다.  
2. **≡** → **API 및 서비스** → **라이브러리** 를 클릭합니다.  
3. 검색창에 **Maps JavaScript API** 를 입력하고 해당 API를 클릭합니다.  
4. **사용** 또는 **Enable** 를 클릭합니다.  
5. **≡** → **API 및 서비스** → **사용자 인증 정보** 를 클릭합니다.  
6. **+ 사용자 인증 정보 만들기** → **API 키** 를 선택합니다.  
7. 만들어진 API 키를 **복사**해 메모장에 저장합니다. (나중에 Cloud Run 환경 변수 `GOOGLE_MAPS_API_KEY` 에 넣습니다.)

# 5. GitHub 가입

SmartBrief 코드는 **GitHub**에 있습니다. GitHub에 가입한 뒤, 그 코드를 **Fork(복사)** 해서 내 계정으로 가져옵니다.

## 5-1. 가입 페이지 접속

1. 브라우저에서 **https://github.com** 을 엽니다.  
2. 오른쪽 위 **Sign up** 를 클릭합니다.  

## 5-2. 계정 정보 입력

1. **이메일 주소** 입력 → **Continue**  
2. **비밀번호** 만들기 (영문+숫자+기호 조합, 15자 이상 권장) → **Continue**  
3. **사용자 이름(Username)** 입력 (예: `myusername`) → **Continue**  
4. **제품 소식 수신** y/n 선택 → **Continue**  
5. **캡차**(퍼즐 등) 풀기  
6. **Create account** 클릭  

## 5-3. 이메일 인증

1. 가입할 때 쓴 **이메일**로 GitHub에서 보낸 **인증 메일**을 엽니다.  
2. 메일 안의 **코드**를 입력하거나 **Verify** 버튼을 클릭합니다.  
3. **Complete setup** 까지 마치면 **가입 완료**입니다.  

---

# 6. SmartBrief 저장소 Fork (내 계정으로 복사)

**Fork**는 다른 사람 저장소를 **내 GitHub 계정에 복사**하는 것입니다. 복사본을 내가 수정·동기화할 수 있습니다.

## 6-1. SmartBrief 저장소 찾기

1. GitHub에 **로그인**한 상태에서  
2. 위쪽 **검색창**에 `smartbrief` 또는 관리자가 알려 준 **저장소 이름** (예: https://github.com/rokafpilot/smartbrief) 을 입력합니다.  
3. **Repositories** 탭에서 해당 저장소를 클릭합니다.  
   - 예: **rokafpilot/smartbrief**  
   
4. 저장소 **메인 페이지**가 열리면 준비된 것입니다.  

## 6-2. Fork 실행

1. 저장소 페이지 **오른쪽 위**를 봅니다.  
2. **Fork** 버튼(포크 모양 아이콘)을 클릭합니다.  
3. **Create fork** (또는 **Fork**) 를 클릭합니다.  
4. 잠시 기다리면 **내 사용자 이름/smartbrief** 형태의 **새 저장소**가 만들어집니다.  
   - 예: `myusername/smartbrief`  
5. 이제 이 주소가 **내 Fork** 주소입니다.  
   - 주소창에 보이는 주소를 메모해 둡니다. 예: `https://github.com/myusername/smartbrief`  

6. **Fork까지 하면 GitHub 준비는 끝입니다.**  
   - 컴퓨터에 코드를 다운로드할 필요 없습니다. 다음 단계에서 **GCP**와 이 Fork를 연결합니다.  

---

# 7. Fork 이후 GCP에서 할 일

Fork만 해 두면 GitHub 쪽 준비는 끝입니다. **실제로 SmartBrief가 동작하려면**, 1~3단계에서 만든 **GCP 프로젝트**와 **내 Fork**를 연결하고, API 키를 서비스에 넣어야 합니다.

## 7-1. Cloud Build 트리거로 “내 Fork” 연결하기

**트리거** = “내 Fork 저장소에 push가 있으면 GCP가 자동으로 빌드·배포해 준다”는 설정입니다.

1. **https://console.cloud.google.com** 에 접속해, 3단계에서 만든 **프로젝트**를 선택합니다.  
2. 왼쪽 메뉴(≡) → **CI/CD** → **Cloud Build** → **트리거** 로 갑니다.  
3. **트리거 만들기** (또는 **Create trigger**) 를 클릭합니다.  
4. **이벤트**: “내 저장소에 푸시할 때” 를 선택합니다.  
5. **소스**:  
   - 처음이면 **저장소 연결** / **GitHub에 연결** 을 눌러 GitHub 로그인 후, **저장소 액세스**에서 **내 Fork**(내 사용자명/smartbrief) 를 허용합니다.  
   - 연결 후 **저장소** 드롭다운에서 **내 Fork** 를 선택합니다.  
     - 예: `내사용자이름/smartbrief` (원본 rokafpilot/smartbrief 가 아니라 **내 계정의 smartbrief**)  
6. **브랜치**: `^main$` 또는 “main” 브랜치를 선택합니다.  
7. **구성**: “Cloud Build 구성 파일” → **cloudbuild.yaml** (저장소 루트) 로 둡니다.  
8. **만들기** (또는 **저장**) 로 트리거를 저장합니다.  

이제 **내 Fork**의 main 브랜치에 push가 있을 때마다 이 GCP 프로젝트에서 자동으로 빌드·배포가 됩니다.

> **중요 — bash 배포 스크립트를 꼭 한 번은 써야 하나요?**  
> **트리거만으로도** “트리거 실행” 또는 “Fork에 push” 시 cloudbuild.yaml이 돌면서 **첫 배포까지** 이론상 가능합니다.  
> 다만 **새로 만든 GCP 프로젝트**에는 아래가 없어서, 트리거 첫 실행 시 **권한 오류(PERMISSION_DENIED)** 나 “저장소 없음” 등으로 실패할 수 있습니다.  
> - 사용 설정이 안 된 API (Cloud Run, Artifact Registry, Cloud Build)  
> - Artifact Registry 저장소(이미지 넣을 곳)  
> - Cloud Build 서비스 계정에 Cloud Run 배포 권한  
> **그래서 안정적으로 하려면**, **deploy 스크립트(deploy_mac xxx.sh)를 한 번 실행**해 두는 것을 권장합니다. 스크립트가 위 설정과 첫 배포를 한 번에 해 주고, 그다음부터는 **트리거만** 써도 됩니다. (스크립트 실행 방법: **md/배포_및_사용자_가이드.md** Part 1 참고. 스크립트 안의 PROJECT_ID, API_KEY 등만 본인 값으로 넣으면 됩니다.)

## 7-2. Cloud Run에 API 키 넣기 (환경 변수)

배포된 SmartBrief가 **Gemini**, **Google 지도** 등을 쓰려면, Cloud Run 서비스에 API 키를 환경 변수로 넣어야 합니다.

1. GCP 콘솔에서 **Cloud Run** → 방금 배포된 **smartnotam3** (또는 해당 서비스 이름) 를 클릭합니다.  
2. **수정 및 새 리비전 배포** 를 클릭합니다.  
3. **변수 및 시크릿** 탭으로 갑니다.  
4. **변수 추가** 로 아래를 넣습니다.  
   - 이름: `GEMINI_API_KEY` / 값: 4단계에서 발급한 **Gemini API 키**  
   - (지도 사용 시) 이름: `GOOGLE_MAPS_API_KEY` / 값: **Google Maps API 키**  
5. **배포** 를 눌러 반영합니다.  

이후에는 트리거가 “내 Fork” 기준으로 자동 배포하고, 위 환경 변수는 그대로 유지됩니다.
