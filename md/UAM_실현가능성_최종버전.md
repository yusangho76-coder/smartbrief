# 실현 가능성 (Solution) - 최종 버전

## 1. 개발 계획 (최종 산출물)

### 기존 개발 기반 활용
본 사업은 **SmartNOTAM**과 **SmartWeather** 시스템을 이미 개발 중인 기술 기반으로 시작하여 빠른 실현 가능성을 보장합니다.

**기존 개발 완료 사항:**
- ✅ **SmartNOTAM**: NOTAM 자동 파싱, AI 기반 번역(Google Gemini API), 항로 기반 필터링, 지리공간 시각화(Google Maps), FIR별 자동 분류
- ✅ **SmartWeather**: 항공 기상 데이터 통합 분석(METAR/TAF, SIGMET, SIGWX 등), 기상 영향도 평가 시스템

**UAM 브리핑 시스템 통합 개발 계획:**

**Phase 1 (0~6개월): 기존 시스템 통합 및 UAM 특화 기능 추가**
- **기존 시스템 통합**: SmartNOTAM + SmartWeather 통합 플랫폼 구축
- **UAM 특화 기능 개발**:
  - 저고도(100~500m) 기상 분석 모델 개발
  - 도심 장애물 데이터 통합 (고층건물, 송전선, 통신탑)
  - 버티포트 정보 연동 모듈 개발
- **브리핑 자동 생성 엔진**: 기존 AI 번역 엔진 확장하여 운항 브리핑 자동 생성 기능 추가
- **테스트 운영**: 제한된 도심 노선 3개, 버티포트 5개에서 파일럿 운영
- **산출물**: UAM 브리핑 시스템 MVP 1건, 파일럿 운영 보고서 1건, 통합 시스템 아키텍처 문서 1건

**Phase 2 (6~12개월): 기능 고도화 및 베타 서비스**
- **저고도 기상 모델 고도화**: 건물 간 바람/난류 예측 모델 개선, 열섬 현상 분석 추가
- **장애물 영향 평가 고도화**: 3D 지형 정보와 운항 경로 교차 분석 정확도 향상
- **위험도 스코어링 시스템**: UAM 운항 제약 모델 고도화
- **베타 서비스 확장**: 노선 10개, 버티포트 15개 확장, 베타 고객 3개사와 협력
- **산출물**: 베타 버전 1건, 베타 운영 보고서 1건, 사용자 평가 보고서 1건, 성능 개선 분석 보고서 1건

**Phase 3 (12~24개월): 상용화 및 확장**
- **UTM 연동**: 무인항공기 교통관리 시스템과의 API 연동 개발
- **다국어 지원**: 영어, 중국어 등 다국어 브리핑 생성 기능 추가
- **맞춤형 브리핑 템플릿**: 운항사별 SOP(Standard Operating Procedure) 맞춤형 브리핑 생성
- **전국 확장**: 전국 주요 노선, 버티포트 30개 이상 지원
- **상용 고객 확보**: UAM 운항사 5개사 이상과 상용 계약 체결
- **산출물**: 상용 버전 1건, 기술 문서화 자료 1건, API 문서 1건, 상용화 성과 보고서 1건

## 2. 개발 창업 아이템의 기능·성능의 차별성 및 경쟁력 확보 전략

### 핵심 차별성

#### 1. 기존 시스템 통합을 통한 빠른 개발 가능성
- **SmartNOTAM과 SmartWeather 통합 활용**: 기존 개발된 핵심 기술을 활용하여 개발 기간 단축 및 기술 검증된 안정적 기반 확보
- **검증된 AI 엔진**: Google Gemini API 기반 번역/생성 엔진이 이미 SmartNOTAM에서 검증됨
- **실증된 데이터 처리**: NOTAM 파싱, 항로 필터링, 지리공간 시각화 기능이 실제 환경에서 검증됨

#### 2. UAM 저고도 운항 특화 설계
- **저고도 기상 분석**: 기존 고고도(FL200 이상) 중심 시스템과 달리 저고도(100~500m) 도심 운항에 특화
  - 건물 간 바람/난류 예측 모델 (기존 시스템 부재)
  - 도심 열섬 현상 분석
  - 국지적 강풍 예측
- **도심 장애물 통합 분석**: 고층건물, 송전선, 통신탑 등 3D 지형 정보와 운항 경로 교차 분석
- **버티포트 운영 연동**: 버티포트 운영 상태, 접근 제한 등 실시간 정보 통합

#### 3. AI 기반 자동 브리핑 생성
- **기존 시스템**: 데이터 단순 제공 (METAR/TAF, NOTAM 원문)
- **본 시스템**: AI가 위험요소를 분석하여 조종사 친화적 문장으로 브리핑 자동 생성
- **SmartNOTAM의 AI 엔진 활용**: 기존 검증된 Gemini 기반 번역 시스템을 브리핑 생성으로 확장

#### 4. 국내 운항 환경 최적화
- **국내 기상 환경 특화**: 국내 도심 기상 패턴 분석 모델
- **국내 규제 반영**: 항공안전법, UAM 관련 규제 자동 반영
- **SWIM 체계 대응**: 국내 SWIM(System Wide Information Management) 체계에 맞춘 정보 관리

### 경쟁력 확보 전략

#### 1. 기술적 경쟁력
- **기존 검증 기술 활용**: SmartNOTAM/SmartWeather 기반으로 기술 리스크 최소화
- **Google Gemini API 기반 고도화**: 이미 검증된 AI 엔진을 브리핑 생성으로 확장
- **저고도 기상 예측 모델 자체 개발**: 기상청 데이터 + 도심 기상 모델링으로 차별화
- **통합 플랫폼 구축**: NOTAM + 기상 + 장애물 정보를 하나의 플랫폼에서 통합 제공

#### 2. 시장 선점 전략
- **국내 UAM 상용화 시기(2025년) 전 MVP 완성**: 2025년 상용화 전에 시스템을 완성하여 시장 선점
- **기존 시스템 대비 빠른 개발**: 기존 시스템 통합으로 개발 기간 단축 (6개월 MVP)
- **파일럿 운영을 통한 초기 고객 확보**: UAM 운영사와 협력하여 초기 고객 및 데이터 확보

#### 3. 파트너십 및 데이터 확보
- **항공공단 협력**: NOTAM 데이터 API 협약 (SmartNOTAM 기존 연동 활용)
- **기상청 협력**: 저고도 기상 데이터 활용 협의
- **UAM 운영사 파트너십**: 파일럿 운영을 통한 실전 데이터 축적 및 피드백 반영
- **버티포트 운영사 협력**: 버티포트 운영 데이터 연동

#### 4. 지속적 개선 체계
- **운항 데이터 축적**: 실제 운항 데이터를 통한 AI 모델 지속 학습 및 개선
- **사용자 피드백 반영**: 베타 운영을 통한 사용자 피드백 기반 기능 개선
- **규제 변화 대응**: UAM 규제 변화에 실시간 대응하는 모듈화된 시스템 구조

#### 5. 글로벌 진출 기반
- **국내 표준 시스템 구축**: 국내에서 성공 사례 구축 후 해외 진출
- **다국어 지원 확장**: 영어, 중국어 등 다국어 브리핑으로 글로벌 시장 대응
- **해외 UAM 운영사 진출**: 국내 기술력 검증 후 해외 시장 진출
