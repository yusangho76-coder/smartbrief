# SmartBrief 배포·사용 가이드 (처음 쓰는 분도 따라 하기)

이 문서는 **두 가지 경우**를 안내합니다.

1. **관리자/개발자**: 처음으로 SmartBrief를 인터넷에 배포하는 분  
2. **일반 사용자**: GitHub에 가입해서 SmartBrief 저장소를 Fork(복사)하는 분  

컴퓨터를 잘 모르셔도 번호 순서대로만 따라 하시면 됩니다.

---

# Part 1. 최초 배포 (관리자·개발자용)

SmartBrief를 Google Cloud Run에 올려서, 누구나 접속할 수 있는 주소(URL)를 만드는 과정입니다.

---

## 1-1. 미리 준비할 것

| 준비물 | 설명 |
|--------|------|
| **컴퓨터** | Mac 또는 Windows (이 가이드는 Mac 기준입니다. Windows는 터미널 대신 명령 프롬프트 또는 PowerShell 사용) |
| **Google 계정** | Gmail 주소. 나중에 Cloud Run·API 키 설정에 사용합니다. |
| **GitHub 계정** | [github.com](https://github.com)에서 무료 가입. 코드 저장소에 접근할 때 필요합니다. |
| **인터넷** | 안정적인 연결이 필요합니다. |

---

## 1-2. 프로그램 설치 (Mac)

### 1) 터미널 열기

- **Mac**: `Spotlight`(⌘+Space) 누르고 `터미널` 또는 `Terminal` 입력 후 엔터  
- 또는 **Finder** → **응용 프로그램** → **유틸리티** → **터미널** 더블클릭  

검은 창에 글자가 보이면 준비된 것입니다.

### 2) Homebrew 설치 (없는 경우)

터미널에 아래를 **한 줄씩** 붙여 넣고 엔터를 누릅니다.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

설치가 끝나면 화면에 나오는 안내대로 **PATH 설정**까지 해 주세요. (예: `echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile` 같은 문장을 복사해 실행)

### 3) Git 설치

터미널에서:

```bash
brew install git
```

### 4) Google Cloud SDK(gcloud) 설치

터미널에서:

```bash
brew install google-cloud-sdk
```

설치 후 **한 번만** 아래를 실행해 로그인합니다.

```bash
gcloud auth login
```

브라우저가 열리면 **Google 계정으로 로그인**합니다.

---

## 1-3. 프로젝트 코드 받기

### 방법 A: GitHub에서 코드를 이미 받아 둔 폴더가 있는 경우

- 그 폴더가 **SmartBrief 프로젝트 루트**입니다.  
- 그 안에 `app.py`, `deploy` 폴더, `templates` 폴더 등이 있어야 합니다.  
- 아래 1-4로 넘어가면 됩니다.

### 방법 B: GitHub에서 처음 받는 경우

1. 브라우저에서 [github.com](https://github.com) 접속 후 로그인  
2. SmartBrief 저장소 주소를 엽니다. (예: `https://github.com/rokafpilot/smartbrief`)  
3. 초록색 **Code** 버튼 클릭 → **HTTPS** 탭에서 주소 복사 (예: `https://github.com/rokafpilot/smartbrief.git`)  
4. 터미널에서 **코드를 둘 폴더**로 이동한 뒤 아래를 실행합니다. (주소는 실제 저장소 주소로 바꾸세요)

```bash
cd ~/Documents
git clone https://github.com/rokafpilot/smartbrief.git
cd smartbrief
```

이제 `smartbrief` 폴더가 프로젝트 루트입니다.

---

## 1-4. 배포 스크립트 설정 (한 번만)

배포 전에 **프로젝트마다 다른 값**을 스크립트에 넣어야 합니다.

1. **프로젝트 루트** 안의 **deploy** 폴더로 갑니다.  
2. **deploy_mac ksj.sh** (또는 사용하는 사람별 스크립트)를 **텍스트 편집기**로 엽니다.  
3. 맨 위 **설정값** 부분만 아래처럼 본인 값으로 수정합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| `PROJECT_ID` | Google Cloud 프로젝트 ID | `smartbrief-490302` |
| `REGION` | 서버 지역 (보통 그대로) | `asia-northeast3` (서울) |
| `REPO` | 이미지 저장소 이름 (보통 그대로) | `smartnotam` |
| `SERVICE` | 서비스 이름 (보통 그대로) | `smartnotam3` |
| `API_KEY` | Gemini API 키 (Google AI Studio에서 발급) | `AIzaSy...` |
| `GRANTEE` | 실행 권한 줄 이메일 | `user:본인이메일@gmail.com` |

4. **저장** 후 편집기를 닫습니다.

**Gemini API 키 받는 방법 (간단)**  
- [Google AI Studio](https://aistudio.google.com/apikey) 접속 → 로그인 → **Create API key** → 키 복사 후 위 `API_KEY` 자리에 붙여 넣기  

**Google Cloud 프로젝트 ID**  
- [Google Cloud Console](https://console.cloud.google.com) 접속 → 상단 프로젝트 선택 시 나오는 **프로젝트 ID** 사용 (새 프로젝트는 **프로젝트 만들기**로 먼저 만듦)

---

## 1-5. 배포 실행

1. 터미널을 엽니다.  
2. **프로젝트 루트**로 이동합니다. (1-3에서 받은 폴더, 예: `cd ~/Documents/smartbrief`)  
3. 아래 명령을 실행합니다. (스크립트 이름은 사용하는 파일명으로 바꾸세요)

```bash
chmod +x "deploy/deploy_mac ksj.sh"
./"deploy/deploy_mac ksj.sh"
```

4. 화면 안내가 나오면:
   - **브라우저 로그인**이 나오면 → Google 계정으로 로그인  
   - **프로젝트 생성/선택**이 나오면 → 사용할 GCP 프로젝트 선택  
   - **Y/n**이 나오면 → `Y` 입력 후 엔터  

5. **에러 없이 끝나면** 마지막에 **Flask 앱 배포 완료: https://...run.app** 같은 주소가 나옵니다. 이 주소가 SmartBrief 접속 주소입니다.

---

## 1-6. 배포 후 확인·추가 설정

### 접속 테스트

- 터미널에 나온 URL을 **브라우저 주소창**에 넣고 엔터  
- SmartBrief 첫 화면이 보이면 성공입니다.  

### Google 지도가 필요할 때

- [Google Cloud Console](https://console.cloud.google.com) → **Cloud Run** → 해당 서비스 클릭  
- **수정 및 새 리비전 배포** → **변수 및 시크릿** 탭  
- **변수 추가** → 이름: `GOOGLE_MAPS_API_KEY`, 값: (지도 API 키)  
- **배포** 클릭  

### 나중에 코드 수정 후 자동 배포 (선택)

- GitHub에 push할 때마다 자동으로 다시 배포되게 하려면 **Cloud Build 트리거**를 설정합니다.  
- 자세한 단계는 **md/CLOUD_BUILD_TRIGGER_SETUP.md** 를 참고하세요.

# Part 2. 사용자: GitHub 가입 후 Fork 하기

SmartBrief **코드를 보고 싶거나, 내 계정에 복사(Fork)해서 쓰고 싶은 분**을 위한 단계입니다.  
배포까지 할 필요 없이 **GitHub에서 가입 → 저장소 찾기 → Fork**만 하면 됩니다.

---

## 2-1. GitHub 가입

1. 브라우저에서 **https://github.com** 접속  
2. 오른쪽 위 **Sign up** 클릭  
3. **이메일** 입력 → **Continue**  
4. **비밀번호** 만들기 (영문+숫자+기호 조합, 15자 이상 권장) → **Continue**  
5. **사용자 이름** (Username) 입력 → **Continue**  
6. **y** 또는 **n** (이메일로 제품 소식 받을지) 선택 → **Continue**  
7. 퍼즐 같은 **캡차** 풀기  
8. **Create account** 클릭  
9. 이메일로 온 **인증 코드** 입력 후 **Verify**  
10. 질문(학생/용도 등)은 원하는 대로 선택하거나 **Skip**  
11. **Complete setup** 끝내면 **가입 완료**입니다.  

---

## 2-2. SmartBrief 저장소 찾기

1. GitHub에 **로그인**한 상태에서  
2. 위쪽 **검색창**에 `smartbrief` 또는 제공받은 **저장소 이름** 입력  
3. **Repositories** 탭에서 해당 저장소 클릭  
   - 예: **rokafpilot/smartbrief**  
4. 저장소 페이지가 열리면 **코드 목록**과 **README**가 보입니다.  

---

## 2-3. Fork(복사) 하기

1. 저장소 페이지 **오른쪽 위**를 봅니다.  
2. **Fork** 버튼(포크 모양 아이콘) 클릭  
3. **Create fork** (또는 **Fork**) 클릭  
4. 잠시 후 **내 계정 이름/smartbrief** 같은 이름의 **새 저장소**가 만들어집니다.  
   - 이제 **내 GitHub 계정**에 SmartBrief 코드 **복사본**이 생긴 것입니다.  
   - 원본(예: rokafpilot/smartbrief)은 그대로 있고, 내 쪽만 수정·추가할 수 있습니다.  

---

## 2-4. Fork한 저장소 확인

1. GitHub 왼쪽 위 **Octocat 로고** 또는 **Your repositories** 클릭  
2. 목록에서 **smartbrief** (내 계정 이름이 붙은 것) 선택  
3. **Code** 버튼을 누르면 **내 Fork 주소**가 보입니다.  
   - 이 주소로 나중에 **Clone**하면 내 컴퓨터에 같은 코드를 받을 수 있습니다.  

---

## 2-5. (선택) 내 컴퓨터에서 Fork한 코드 실행하기

코드를 **내 컴퓨터에서 실행**해 보고 싶다면:

1. **Part 1**의 1-2(Git 설치), 1-3 **방법 B**처럼 **Clone**할 때, 주소만 **내 Fork 주소**로 바꿉니다.  
   - 예: `git clone https://github.com/내계정/smartbrief.git`  
2. 받은 폴더로 들어가서 **Python 환경**을 만들고 **requirements.txt**로 패키지 설치 후 **app.py** 실행하면 됩니다.  
   - 자세한 실행 방법은 프로젝트 **README** 또는 관리자에게 문의하세요.  

**배포(Cloud Run에 올리기)** 는 Part 1을 진행하는 사람만 하면 되고, **일반 사용자는 Fork만 해도** 코드 확인·보관 목적에는 충분합니다.
