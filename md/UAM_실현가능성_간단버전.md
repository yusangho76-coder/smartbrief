# 실현 가능성 (Solution) - 간단 버전

## 1. 개발 계획 (최종 산출물)

**사업기간 내 개발 계획:**
- **0~6개월 (MVP)**: UAM 브리핑 시스템 MVP 개발 완료
  - 기본 NOTAM/기상 통합 분석 및 브리핑 자동 생성 기능
  - 제한된 노선 3개, 버티포트 5개에서 파일럿 운영
  - 산출물: MVP 1건, 파일럿 운영 보고서 1건

- **6~12개월 (베타)**: 기능 고도화 및 베타 서비스 런칭
  - 저고도 기상/장애물 모델 고도화, 위험도 스코어링 개선
  - 노선 10개, 버티포트 15개 확장, 베타 고객 3개사
  - 산출물: 베타 버전 1건, 운영 보고서 1건

- **12~24개월 (상용화)**: 상용 버전 출시 및 확장
  - UTM 연동, 다국어 지원, 맞춤형 브리핑 템플릿
  - 전국 주요 노선, 버티포트 30개 이상, 상용 고객 5개사 이상
  - 산출물: 상용 버전 1건, 기술 문서 1건, API 문서 1건

## 2. 차별성 및 경쟁력 확보 전략

**핵심 차별성:**
1. **UAM 저고도 운항 특화**: 기존 고고도 항로 중심 시스템과 달리 저고도(100~500m) 도심 운항에 특화된 위험요소 분석
2. **AI 기반 자동 브리핑 생성**: 데이터 단순 제공이 아닌 AI가 위험요소를 분석하여 조종사 친화적 문장으로 브리핑 자동 생성
3. **도심 장애물 통합 분석**: 고층건물, 송전선 등 3D 지형 정보와 운항 경로를 교차 분석하여 기존 시스템에서 제공하지 않는 정보 제공

**경쟁력 확보 전략:**
- Google Gemini API 기반 고도화된 AI 엔진 활용 및 저고도 기상 예측 모델 자체 개발
- 항공공단, 기상청과의 데이터 협약 및 UAM 운영사 파일럿 운영을 통한 실전 데이터 축적
- 국내 UAM 상용화 시기(2025년) 전 MVP 완성으로 시장 선점 및 글로벌 진출 기반 마련
